Requirements

  1. Data are in the smart cities domain.

    The data we will be using is about the city of San Francisco.
    This data is obtained from the webpage: https://data.sfgov.org/, which is a "hub"
    for all the open data available at the city of San Francisco.
    With all the data that is available, we can assume that this city is indeed
    in the smart cities domain

  2. Data are available as a CSV file

    All the data is available in an array of different formats, such as:
    CSV, JSON, XML and RDF.

  3. Data have an open license so it can be published

    The webpage has a "Terms of Use", which specifies that all the data in it is available
    to use freely.
    Some datasets have an additional license, but almost all of them are open licenses.

  4. Data can be easily linked with generic real-world entities

    All the data available is created by third-parties, but is linked to the city of
    San Francisco, so all of it is linked directly to a real-world entity.

  5. Documentation exists for the data (optional)

    Because we have the provider of each dataset, we can always seek more information
    outside the https://data.sfgov.org/ webpage.

  6. Data can come from multiple data sources (optional)

    All the data comes from the same webpage, but each one is propvided by different
    publishers.
