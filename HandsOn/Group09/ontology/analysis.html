1 Data sets analysis

1.1 alojamiento.csv

1.1.1 Headers, columns or filds definition

In this CSV file we have the next headers (or columns), each header will be defined:

  • id : String -> The id of the "alojamiento", actually is an URL that points to the web site info.
  • title : String -> The name of the "Alojamiento"
  • description : String -> A brief description of the "alojamiento", some entries use it as contact field
  • streetAddress : String -> The address of the "alojamiento"
  • postalCode : Int -> zip code
  • addressLocality : String -> City Name
  • Telefonos : Int -> Telephone number
  • Fax : String -> Fax number
  • email : String -> Contact email
  • url : String -> Alojaiento's website
  • image : String -> A path to the site's image (It's usual to find it empty)
  • logo : String -> A path to the site's logo (It's usual to find it empty)
  • socioHoreca : String (True or False isomorphism) -> denotes if this site is member of the club Horeca
  • accesibilidad : String -> Additional information about movility in the place, usually a HTML div.
  • commet : String -> uncategorized information comment
  • categtoria : String -> Place's category
  • habitaciones : Int -> Number of bedrooms
  • camas : Int -> Number of beds
  • geometry : duple(Int) -> Coordinates in the map
  • lastUpdated : String -> A last information update date
  • link : String -> The URI or path of the Alojamiento inside the www.zaragoza.es website

1.1.2 Data schema

A completely valid data schema (accordig to: http://digital-preservation.github.io/csv-schema/csv-schema-1.0.html) could be: 

version 1.0
@totalColumns 21
id: notEmpty and String
title: notEmpty and String
description: String or ""
streetAddress: notEmpty and String
postalCode: notEmpty and Int
addressLocality: notEmpty and String
Telefonos: notEmpty and String 
Fax: String or ""
email: String or ""
url: notEmpty and String
image: String or ""
logo: String or ""
socioHoreca: notEmpty and is(True) or is(False)
accesibilidad: String or ""
commet: String or ""
categtoria: notEmpty and String
habitaciones: notEmpty and Int
camas: notEmpty and Int
geometry: duple(Float) or ""
lastUpdated: notEmpty and String
link : notEmpty and String

1.2 restaurante.csv

1.2.1 Headers, columns or filds definition

In this CSV file we have the next headers (or columns), each header will be defined:

  • id : Stirng -> The id of the restaurant, actually is an URL
  • title: String -> The name of the restaurant
  • streetAddress : String -> The address of the restaurant
  • postalCode : Int -> zip code
  • addressLocality : String -> City Name
  • tel : Int -> Telephone number
  • email : String -> Contact email
  • url : String -> restaurants's website
  • image : String -> A path to the site's image (It's usual to find it empty)
  • logo : String -> A path to the site's logo (It's usual to find it empty)
  • restauranteEnHotel : String (True or False isomorphism) -> defines if the restaurant is inside of an Hotel
  • socioHoreca : String (True or False isomorphism) -> denotes if this site is member of the club Horeca
  • sinGluten : String (True or False isomorphism) -> Denotes if the restaurant serves food gluten free
  • fincaCatering : String (True or False isomorphism) -> Information about Finca Catering
  • servesCuisine : String -> Information about Serves Cuisine
  • accesibilidad : String -> Additional information about movility in the restaurant
  • comment : String -> uncategorized information comment
  • tenedores : Int -> The category of the restaurant in "Tenedores"
  • capacidad : Int -> Capacity of the restaurant
  • geometry : duple(Float) -> Coordinates in the map
  • lastUpdated : String -> A last information update date
  • description : String -> A brief description of the restaurant
  • link : String -> The URI or path of the restaurant inside the www.zaragoza.es website

1.2.2 Data schema

Again, accordig to: http://digital-preservation.github.io/csv-schema/csv-schema-1.0.html, the schema could be: 

version 1.0
@totalColumns 22
id: notEmpty and String
title: notEmpty and String
streetAddress: notEmpty and String
postalCode: notEmpty and Int
addressLocality: notEmpty and String
tel: Int or ""
email: String or ""
url: String or ""
image: String or ""
logo: String or ""
restauranteEnHotel: notEmpty and is(True) or is(False)
socioHoreca: notEmpty and is(True) or is(False)
sinGluten: empty
fincaCatering: String or ""
servesCuisine: is("Sin Gluten") or ""
accesibilidad: String or ""
comment: String or ""
tenedores: Int or ""
capacidad: Int or ""
geometry: duple(Float)
lastUpdated: notEmpty and String
description: notEmpty and String
link: notEmpty and String

1.3 monumento.csv

1.3.1 Headers, columns or filds definition

In this CSV file we have the next headers (or columns), each header will be defined:

  • id: String -> The id of the monument, actually is an URL
  • title: String -> The name of the monument
  • description: String -> A brief description of the monument
  • estilo: String -> The architecture style of the monument
  • address: String -> The address of the monument
  • horario: String -> The timetable of the monument
  • phone: Int -> Contact number phone of the monument
  • fax: Int -> Contact number fax of the monument
  • datacion: String -> Age of the monument
  • uso: String -> Description of the monument usage
  • pois: String -> Points of interest of the monument
  • datos: String -> Uncategorized information of the monument
  • price: String -> Taxes of the monument
  • visita: String -> Kind of visit
  • image: String -> A picture of the monument
  • top: String (True of False isomorphism) -> Undefined
  • foursquare: String -> foursuare profile
  • lastUpdated: String -> A last information update date
  • geometry: duple(Float) -> Coordinates in the map

1.4 estacion-bicicletas.csv

1.4.1 Headers, columns or filds definition

2 Resource Name Strategy

El dominio para todos los recursos sera el siguiente: www.websemantica.com/ Este dominio es comun para todos nuestros recuersos

1.1 RNS Alojamientos

  • Como Path individual a un alojamiento seria: www.websemantica.com/grupo09/alojamiento/<nomnbre-alojamiento> e.g: www.websemantica.com/grupo09/alojamiento/hotel-delicias

    Otra forma de hacerlo seria poniendo una referencia, por ejemplo un id www.websemantica.com/grupo09/alojamiento/<alojamiento-id> e.g: www.websemantica.com/grupo09/alojamiento/1068

  • Como path para los terminos se usa el # para acceder al campo de interes: www.websemantica.com/grupo09/alojamiento/<nomnbre-alojamiento>#<nombre-del-campo> www.websemantica.com/grupo09/alojamiento/<alojamiento-id>#<nombre-del-campo> e.g: www.websemantica.com/grupo09/alojamiento/hotel-delicias#Camas www.websemantica.com/grupo09/alojamiento/1068#Camas

1.2 RNS Estacion-bicicletas

  • Como Path individual a una estacion de bicicletas seria: www.websemantica.com/grupo09/estacion-bicicletas/<id-estacion> e.g: www.websemantica.com/grupo09/estacion-bicicletas/101
  • Como path para los terminos se usa el # para acceder al campo de interes: www.websemantica.com/grupo09/estacion-bicicletas/<id-estacion>#<nombre-del-campo> e.g: www.websemantica.com/grupo09/estacion-bicicletas/10#bicisDisponibles

1.3 RNS Monumentos

  • Como Path individual a un monumento seria: www.websemantica.com/grupo09/monumento/<nomnbre-monumneto> e.g: www.websemantica.com/grupo09/monumento/casa-de-los-morales

    Otra forma de hacerlo seria poniendo una referencia, por ejemplo un id www.websemantica.com/grupo09/monumentos/<id> e.g: www.websemantica.com/grupo09/alojamientos/26

  • Como path para los terminos se usa el # para acceder al campo de interes: www.websemantica.com/grupo09/monumento/<nomnbre-monumento>#<nombre-del-campo> www.websemantica.com/grupo09/monumento/<id-monumento>#<nombre-del-campo> e.g: www.websemantica.com/grupo09/monumento/casa-de-los-morales#description www.websemantica.com/grupo09/monumento/26#description

1.4 RNS Poste

  • Como Path individual para un poste seria: www.websemantica.com/grupo09/poste/<id-poste> e.g: www.websemantica.com/grupo09/poste/798
  • Como path para los terminos se usa el # para acceder al campo de interes: www.websemantica.com/grupo09/poste/<id-poste>#<nombre-del-campo> e.g: www.websemantica.com/grupo09/poste/798#geometry

1.5 RNS Restaurante

  • Como Path individual a un restaurante seria: www.websemantica.com/grupo09/restaurante/<nomnbre-restaurante> e.g: www.websemantica.com/grupo09/restaurante/guetaria

    Otra forma de hacerlo seria poniendo una referencia, por ejemplo un id www.websemantica.com/grupo09/restaurantes/<id> e.g: www.websemantica.com/grupo09/alojamientos/61

  • Como path para los terminos se usa el # para acceder al campo de interes: www.websemantica.com/grupo09/restaurante/<nomnbre-restaurante>#<nombre-del-campo> www.websemantica.com/grupo09/restaurante/<id-restaurante>#<nombre-del-campo> e.g: www.websemantica.com/grupo09/restaurante/guetaria#streetAddress www.websemantica.com/grupo09/restaurante/26#streetAddress
